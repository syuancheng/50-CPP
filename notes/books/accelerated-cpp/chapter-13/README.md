# Chapter 13：继承、动态绑定与多态值

`Core` 表示本科生，`Grad` 在其基础上增加论文成绩。通过基类引用或指针调用虚函数时，实际对象类型决定调用哪个重写函数，这就是动态绑定。

## 构造与析构

构造派生对象时：

1. 为完整对象取得存储；
2. 先构造基类子对象；
3. 再按声明顺序构造成员；
4. 最后执行派生类构造函数体。

析构顺序相反。若对象可能通过基类指针删除，基类析构函数必须为 `virtual`。

## 克隆与值语义

`Student_info` 持有 `Core*`，但对外表现为可复制的值。复制时不能只复制地址，否则两个对象会重复删除同一资源；它通过虚 `clone()` 创建实际动态类型的副本。

赋值先完成克隆，再替换旧指针。若克隆抛出异常，原对象保持不变，这是强异常保证。
