# 学习路线

路线按依赖关系排列：后面的主题建立在前面的对象模型和生命周期之上。

## 第一阶段：写出确定行为的单线程程序

1. 类型、初始化、表达式、控制流与函数。
2. `string`、`vector`、范围 `for` 和基本算法。
3. 引用、指针、`const`、作用域与生命周期。
4. 类、构造函数、不变量、封装与值语义。
5. 输入输出、文件和错误处理。

完成标准：能写一个多文件控制台程序；严格警告下可编译；不依赖裸 `new` / `delete`。

## 第二阶段：掌握 C++ 的资源与抽象机制

1. RAII、智能指针、Rule of Zero/Five。
2. 继承、虚函数、对象切片和虚析构。
3. 函数模板、类模板、lambda 与泛型算法。
4. 编译、链接、头文件、库和 CMake target。
5. 单元测试、调试器和 Sanitizer。

完成标准：能解释每个资源的所有者和生命周期；能定位编译错误与链接错误的区别。

## 第三阶段：工程必需能力

1. 容器复杂度、迭代器失效和性能测量。
2. 异常安全、API 设计与依赖管理。
3. 并发、数据竞争、锁与条件变量。
4. 网络、数据库、序列化等领域库。

并发不是“会调用 `std::thread`”就算完成。应先理解 C++ 内存模型，并能清楚说明共享状态由谁保护。

## 练习方法

每个知识点用四步闭环：

1. 写出对象、资源和不变量。
2. 写最小正确示例。
3. 写一个反例并解释失败原因。
4. 用警告、测试、Sanitizer 或性能基准验证。

先预测结果，再编译运行；无法预测时说明模型仍不完整。
